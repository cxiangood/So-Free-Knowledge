# SoFree Knowledge 飞书主动知识Agent

**担心灵感迸发沉群底？害怕黑话太多看不懂？别怕，SoFree 稳稳接住你！**

> **SoFree = So Free Knowledge**：让知识保持自由流动，不被信息洪流裹挟，不因黑话门槛而阻断，不因消息沉底而错失。我们希望每一条有价值的信息都能被及时看见、被准确理解、被快速行动，这就是 SoFree 所定义的“如此自由”。

不同于传统**被动记录**的知识库工具或**被动问答**的聊天机器人，SoFree 以大模型驱动 “知识聚合、黑话检测、价值检测” 三大模块，持续从飞书消息中提炼重点、沉淀术语、实时识别价值并**主动分发**给用户，让每条关键信息都被正确的人看见、被理解、被执行。

项目围绕三个“分子模块”构建完整闭环：**知识聚合、黑话检测、价值检测**。三者均支持定时或主动触发（价值检测支持实时监听），共同解决“消息太散、术语太黑、价值被淹没”的问题。

我们的系统将飞书中的零散消息统一纳入同一条智能流水线：自动拉取上下文 → LLM 提炼与判断 → 结构化卡片分发或词典沉淀，让有价值的信息不再停留在聊天记录里。

## ✨ 三大核心能力

### 🧩 知识聚合：把分散消息整理成可读卡片
**解决痛点**：关键信息散落在多个群聊与文档中，用户很难在高频消息流里快速拼出完整上下文。

**实现能力**：
- 通过定时或主动触发，聚合用户关注范围内的文本与消息
- 基于 LLM 对多条消息去重、归并、摘要，形成结构化知识卡片
- 自动推送聚合结果，帮助用户低成本获取“这段时间最重要的信息”
- 保留来源上下文，便于快速追溯原始讨论

**核心价值**：把“读大量消息”变成“看一张卡片”，显著降低信息筛选与同步成本。

### 🧠 黑话检测：从群聊中挖掘术语并沉淀到飞书词典
**解决痛点**：行业黑话、项目简称、内部代号高频出现但缺少统一解释，造成跨团队沟通障碍与新人理解门槛。

**实现能力**：
- 通过定时或主动触发，拉取用户账号下群聊消息进行术语抽取
- 使用 LLM 识别高价值黑话/关键词并生成标准化释义
- 自动录入飞书词典（Lingo）并支持持续增量更新
- 结合上下文做术语消歧，减少误收录与噪声词

**核心价值**：把“口口相传的隐性黑话”沉淀为“可检索、可复用、可传承”的组织词汇资产。

### ⚡ 价值检测：实时捕捉高价值信息并主动分发
**解决痛点**：关键任务、重要决策与灵感往往转瞬即逝，容易被后续聊天淹没，错过最佳响应时机。

**实现能力**：
- 实时监听聊天流，持续检测高价值信息片段
- 利用 LLM 判断内容价值类型（如关键任务、可执行行动、灵感线索）
- 自动整理为价值卡片并按角色/关联人精准推送
- 建立反馈回路，持续优化价值判断与推送命中率

**核心价值**：将“被动回看聊天记录”升级为“实时发现并及时行动”，提升团队执行效率与机会捕获能力。

## 🏗️ 技术架构
我们采用分层可扩展架构，兼顾算法效果和工程稳定性：

### 1. 数据接入层
- 多渠道数据采集：飞书消息、文档、知识库、用户行为全链路数据接入
- 实时+离线双模式：支持实时消息流处理和离线批量处理
- 数据清洗过滤：自动过滤系统消息、机器人通知、无效@等噪音数据

### 2. 知识结构化层
- 知识升维能力：基于大模型的文本理解与结构化能力，将非结构化消息转化为标准化知识卡片
- 知识存储体系：支持词典库、知识库、用户画像库多维度存储
- 质量校验机制：LLM评审+规则校验，确保结构化知识的准确性

### 3. 个性化决策层
- 用户画像引擎：基于多维度数据自动构建和更新用户兴趣画像
- 混合召回引擎：关键词匹配+双塔语义召回的混合架构，兼顾冷启动效果和数据积累后的效果优化
- 排序策略：支持规则排序、模型排序、人工干预多层排序机制
- 知识路由策略：支持大模型的智能路由，确保不同类型的知识被送到最合适的处理模块

### 4. 交互触达层
- 飞书原生交互：完全适配飞书生态，支持消息回复、卡片推送、bot交互等多种形式
- 多场景推送：支持知识推荐卡、兴趣确认卡、黑话解释等多种推送类型
- 用户反馈体系：支持查看、评论等反馈能力，形成算法优化闭环

## 🚀 快速开始

### 环境要求
- Python 3.13+
- OpenClaw 运行环境
- 飞书企业自建应用权限
- 大模型API服务（支持火山引擎/ OpenAI/ Anthropic/ Claude/通义千问等） 

### 部署步骤
1. 克隆项目：
```bash
git clone https://github.com/cxiangood/So-Free-Knowledge.git
cd So-Free-Knowledge
```

2. 安装依赖与CLI工具：
```bash
pip install -r requirements.txt
pip install -e sofree-knowledge-cli
```

3. 配置环境变量（最好配置在上级目录的.env中）：
```bash

# 飞书应用相关配置
export FEISHU_APP_ID="你的飞书OpenClaw应用ID"
export FEISHU_APP_SECRET="你的飞书OpenClaw应用密钥"
export APP_ID="你的飞书消息价值检测助手应用ID"
export SECRET_ID="你的飞书消息价值检测助手应用密钥"
# 管理员Open ID配置（用于接收任务失败的fallback消息）
export TASK_PUSH_FALLBACK_RECEIVE_ID="管理员用户的open_id"
# 大模型相关配置
export LLM_API_KEY="你的大模型API密钥"
export LLM_MODEL_ID="你的大模型ID"
export LLM_BASE_URL="你的大模型服务URL"
#Hugging Face Token配置（用于快速加载Embedding模型）
export HF_TOKEN="你的Hugging Face Token"

```

4. 启动飞书消息价值检测服务：
```bash
python start_insight.py
```

### 常用命令使用
#### 知识推荐推送（OpenClaw使用）
```bash
# 主动推送知识+兴趣混合卡片给用户
sofree-knowledge assistant recommend --push

# 只推送知识摘要卡片（不推兴趣内容）
sofree-knowledge assistant recommend --push --no-push-interest-card

# 只推送兴趣 digest 卡片（不推知识摘要）
sofree-knowledge assistant recommend --push --no-push-summary-card

# 配置定时推送（在用户profile中设置推送时间）
sofree-knowledge assistant set-profile --weekly-brief-cron "0 9 * * MON" --nightly-interest-cron "0 21 * * *"
```

#### 知识发现与词典同步
```bash
# 从聊天记录中抽取关键词上下文
sofree-knowledge lingo extract-contexts --keywords "关键词1,关键词2" --messages-file ./messages.json

# 从本地词典文件批量同步到本地+飞书Lingo
sofree-knowledge lingo sync-from-file --input-file ./dictionary.json
```

#### 困惑检测与响应
```bash
# 从聊天记录中检测困惑候选
sofree-knowledge confused detect-candidates --messages-file ./messages.json
```
