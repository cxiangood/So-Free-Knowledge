# SoFree Knowledge 飞书主动知识Agent
> 担心灵感迸发沉群底？害怕黑话太多看不懂？别怕，SoFree 稳稳接住你！

SoFree Knowledge 是嵌入飞书的知识收集与分发智能Agent，致力于在群聊与碎片化信息洪流中，打捞被淹没的价值内容，打破"灵感沉底、知识断层、信息错位"的困境，让每一个有价值的想法都不被忽略，让组织知识真正自由流动。

飞书官方提供了强大的群聊、文档、知识库、词典等基础设施，但始终缺少一个完整的知识运营闭环：**知识散落在聊天中没人发现、用户看到消息看不懂没人解释、有价值的内容找不到正确的受众**。

我们的系统补上了这关键的三步：从聊天中自动发现知识 → 用大模型结构化知识 → 按用户画像主动分发知识，打造了一个完整的飞书原生主动知识运营体系。

---

## ✨ 三大核心能力

### 🔍 智能知识发现：把隐性知识变成显性资产
**解决痛点**：群聊里大量业务黑话、项目代号、行业缩写没有统一解释，新人看不懂，老员工记不住，官方词典需要手动维护成本极高。

**实现能力**：
- 自动从聊天上下文抽取候选专业术语、业务黑话
- 基于LLM工作流自动评审：识别有效术语、过滤无效内容、自动生成标准释义
- 双向同步：自动回写到本地知识库和飞书官方Lingo词典
- 增量更新：持续从新的聊天内容中发现新的知识项

**核心价值**：不需要人工干预，自动把散落在对话中的隐性知识变成组织可复用的结构化知识资产。

### 🎯 个性化知识分发：让知识在正确时间到达正确的人
**解决痛点**：重要消息淹没在群聊海洋里，用户每天刷几百条消息也找不到对自己有价值的内容，官方只有被动搜索没有主动推送能力。

**实现能力**：
- 多源数据采集：自动聚合群消息、文档更新、知识库变动、用户访问记录
- 用户画像构建：基于用户的岗位、聊天内容、历史行为自动生成个性化兴趣标签
- 智能召回引擎：冷启动阶段基于关键词匹配快速上线，数据积累后自动切换到双塔语义召回模型，推荐准确率相比纯关键词提升29.5%
- 多层校验机制：经过规则过滤、模型打分、人工复核多层校验，确保推送内容质量
- 交互式卡片推送：通过飞书机器人主动推送个性化知识卡片，支持用户反馈优化推荐效果

**核心价值**：把知识管理从"人找知识"的被动模式，升级为"知识找人"的主动模式，大幅降低信息获取成本。

### 💡 实时困惑响应：让知识解释无处不在
**解决痛点**：用户在群聊中表现出困惑（问号、追问、听不懂的黑话）时，没有人及时解释，导致沟通效率低下，信息传递损耗。

**实现能力**：
- 智能困惑检测：自动识别聊天中的高疑惑回复、提问式对话、术语密集场景
- 上下文感知解释：结合聊天上下文和知识库内容，自动生成易懂的解释内容
- 多形式响应：支持 inline 消息回复、单独卡片推送、补充词典条目等多种响应方式
- 闭环优化：用户的提问和反馈自动回流到知识库，持续完善知识体系

**核心价值**：让知识服务从"人找答案"升级为"答案找人"，大幅提升团队沟通效率。

## 🏗️ 技术架构
我们采用分层可扩展架构，兼顾算法效果和工程稳定性：

### 1. 数据接入层
- 多渠道数据采集：飞书消息、文档、知识库、用户行为全链路数据接入
- 实时+离线双模式：支持实时消息流处理和离线批量处理
- 数据清洗过滤：自动过滤系统消息、机器人通知、无效@等噪音数据

### 2. 知识结构化层
- LLM工作流引擎：借鉴LangChain设计思想，实现"上下文抽取→Prompt构建→LLM推理→结构化解析→知识入库"的全链路可编排工作流
- 知识存储体系：支持词典库、知识库、用户画像库多维度存储
- 质量校验机制：多轮LLM评审+规则校验，确保结构化知识的准确性

### 3. 个性化决策层
- 用户画像引擎：基于多维度数据自动构建和更新用户兴趣画像
- 混合召回引擎：关键词匹配+双塔语义召回的混合架构，兼顾冷启动效果和数据积累后的效果优化
- 排序策略：支持规则排序、模型排序、人工干预多层排序机制
- AB测试框架：支持不同策略的效果对比和灰度发布

### 4. 交互触达层
- 飞书原生交互：完全适配飞书生态，支持消息回复、卡片推送、bot交互等多种形式
- 多场景卡片：支持知识推荐卡、术语解释卡、兴趣确认卡等多种卡片类型
- 用户反馈体系：支持点赞、不感兴趣、纠正错误等反馈能力，形成算法优化闭环

## 🚀 快速开始

### 环境要求
- Python 3.13+
- OpenClaw 运行环境
- 飞书企业自建应用权限
- 大模型API服务（支持OpenAI/ Anthropic/ Claude/通义千问等）

### 部署步骤
1. 克隆项目：
```bash
git clone https://github.com/cxiangood/So-Free-Knowledge.git
cd So-Free-Knowledge
```

2. 安装依赖与CLI工具：
```bash
pip install -r requirements.txt
pip install -e sofree-knowledge-cli
```

3. 配置环境变量（最好配置在上级目录的.env中）：
```bash
export FEISHU_APP_ID="你的飞书OpenClaw应用ID"
export FEISHU_APP_SECRET="你的飞书OpenClaw应用密钥"
export LLM_API_KEY="你的大模型API密钥"
export LLM_MODEL_ID="你的大模型ID"
export LLM_BASE_URL="你的大模型服务URL"
```

4. 启动Bot监听服务：
```bash
python main.py
```

### 常用命令使用
#### 知识推荐推送（OpenClaw使用）
```bash
# 主动推送知识+兴趣混合卡片给用户
sofree-knowledge assistant recommend --push

# 只推送知识摘要卡片（不推兴趣内容）
sofree-knowledge assistant recommend --push --no-push-interest-card

# 只推送兴趣 digest 卡片（不推知识摘要）
sofree-knowledge assistant recommend --push --no-push-summary-card

# 配置定时推送（在用户profile中设置推送时间）
sofree-knowledge assistant set-profile --weekly-brief-cron "0 9 * * MON" --nightly-interest-cron "0 21 * * *"
```

#### 知识发现与词典同步
```bash
# 从聊天记录中抽取关键词上下文
sofree-knowledge lingo extract-contexts --keywords "关键词1,关键词2" --messages-file ./messages.json

# 从本地词典文件批量同步到本地+飞书Lingo
sofree-knowledge lingo sync-from-file --input-file ./dictionary.json
```

#### 困惑检测与响应
```bash
# 从聊天记录中检测困惑候选
sofree-knowledge confused detect-candidates --messages-file ./messages.json
```
